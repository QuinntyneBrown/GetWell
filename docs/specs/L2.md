# Get Well App — L2 Requirements

## L1-1: Welcome / Onboarding Screen

### L2-1.1: App Branding & Mission Statement
The welcome screen shall display the Get Well logo/name and a brief mission tagline ("Free. Anonymous. Always here.").
- **Acceptance Criteria:** Logo/app name is prominently displayed; tagline is visible below it; warm, calming visual tone.

### L2-1.2: Mode Selection
The welcome screen shall present two clear, equally-weighted options: "Get Well" and "Get Well Faith" with brief descriptions of each mode.
- **Acceptance Criteria:** Two distinct buttons/cards for each mode; each has a short description; neither mode is presented as primary over the other.

### L2-1.3: Privacy Assurance
The welcome screen shall include a visible privacy statement ("No accounts. No data stored. Completely anonymous.").
- **Acceptance Criteria:** Privacy statement is visible on the welcome screen without scrolling.

---

## L1-2: Chat Interface — Get Well Mode

### L2-2.1: Chat Message Layout
The chat screen shall display a conversation thread with user messages and AI responses in a standard chat bubble layout.
- **Acceptance Criteria:** User messages appear on the right in accent color; AI responses appear on the left with avatar; messages are in chronological order with clear visual distinction.

### L2-2.2: Message Input Area
The chat screen shall provide a text input field with a send button at the bottom of the screen.
- **Acceptance Criteria:** Input field is always visible; send button is clearly tappable; placeholder text says something warm like "How are you feeling?".

### L2-2.3: Chat Header
The chat screen shall display a header with the mode name ("Get Well"), a calming indicator, and navigation back to mode selection.
- **Acceptance Criteria:** Header shows mode name; back navigation is present; visual tone is warm and calming.

---

## L1-3: Chat Interface — Get Well Faith Mode

### L2-3.1: Faith-Based Chat Layout
The faith chat screen shall display conversation with AI responses that can include scripture references and prayer prompts, visually distinguished from regular text.
- **Acceptance Criteria:** Scripture references are styled distinctly (e.g., italic or highlighted block); prayer prompts are visually identifiable; overall layout mirrors Get Well mode for consistency.

### L2-3.2: Faith Mode Header & Visual Identity
The faith chat screen shall have a distinct visual identity (e.g., cross icon, different accent color) while maintaining the same warm, calming feel.
- **Acceptance Criteria:** Header shows "Get Well Faith" with a faith-related icon; color accent differentiates from general mode; overall warmth is maintained.

### L2-3.3: Faith Mode Input Area
The faith chat screen shall provide the same input experience as Get Well mode with faith-appropriate placeholder text.
- **Acceptance Criteria:** Input field with placeholder like "Share what's on your heart..."; send button present; consistent with general mode layout.

---

## L1-4: Crisis Detection & Resource Referral

### L2-4.1: Crisis Resource Banner
When crisis signals are detected, the chat shall display a compassionate banner with crisis hotline numbers and text line resources.
- **Acceptance Criteria:** Banner is visually prominent but not alarming; includes 988 Suicide & Crisis Lifeline and Crisis Text Line; includes tap-to-call/text actions.

### L2-4.2: Crisis Resource Detail Screen
The app shall provide a dedicated crisis resources screen accessible from the banner or navigation.
- **Acceptance Criteria:** Lists multiple crisis resources with descriptions; each resource has a direct action (call/text); warm, supportive tone throughout.

---

## L1-5: Donation Screen

### L2-5.1: Donation Options
The donation screen shall present preset donation amounts and a custom amount option.
- **Acceptance Criteria:** At least 3 preset amounts displayed; custom amount input available; clear "Donate" CTA button.

### L2-5.2: Mission Reinforcement
The donation screen shall include a brief statement about how donations support the mission.
- **Acceptance Criteria:** Short mission statement visible; 501(c)(3) status mentioned; no-pressure tone.

---

## L1-6: Settings / About Screen

### L2-6.1: About Section
The settings screen shall display information about Get Well's mission, the nonprofit status, and the team.
- **Acceptance Criteria:** Mission statement displayed; 501(c)(3) status visible; contact information (getwell.help) shown.

### L2-6.2: Privacy & Data Info
The settings screen shall clearly state that no personal data is collected or stored.
- **Acceptance Criteria:** Privacy statement prominently displayed; explains anonymous session handling.

### L2-6.3: App Navigation
The settings screen shall provide links to crisis resources, donation page, and mode switching.
- **Acceptance Criteria:** Navigation links to all major sections are present and tappable.
